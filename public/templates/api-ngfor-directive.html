Creates embedded views based on a template by iterating over a model.

Selector
[ngFor]

Using ngFor

Quick Reference
//Creates embedded views with single root element (here, some-element). <some-element *ngFor="let item of items">...</some-element> // Interates over the contents of a template <template ngFor let-item [ngForOf]="items" let-i="index" [ngForTrackBy]="trackByFn"> <some-element>...</some-element><other-element>...</other-element> </template>
ngFor Properties
ngForTrackBy specifies a track by function that can be used to determine model
ngForTemplate specifies a custom TemplateRef to be used for view instantiation

Description link

The NgFor directive is designed to render a list of items into the view and keep that rendered list up to date when the contents of that list change.

NgFor can only iterate over array-like values (key/value maps are not supported). The HTML snippet below shows how ngFor is used to repeat a div element:

Repeat a div element

<div *ngFor=”let item of items.name”> {{ item.value }} </div>

The items variable (which is an array-like value) must be defined on the component that houses the template which contains this ngFor code.

The items variable

class NgForDemoComponent { public items = [ { value: ‘hello’, id: 1 }, { value: ‘hi’, id: 2 }, { value: ‘hey’, id: 3 } ]; }

If a value is removed, moved or inserted into the list then the rendered content on screen will update automatically.

Reordering / Sorting link

To change the order of how the content within the list of is displayed a pipe can be used within the *ngFor expression value.

Reordering / Sorting

<div *ngFor=”let item of items | orderBy: item.id”> {{ item.value }} </div>

Filtering Data link

Much like how reordering/sorting works with ngFor, filtering content behaves the same way and is available using pipes. Use filter: to filter content within NgFor:

Filtering Data

<!-- some variable is a bound to search field --> <input type=”search” #searchField /> <div *ngFor=”let item of items | filter: searchField.value”> {{ item.value }} </div>

Multiple Root Elements Example

The template element allows you to group several elements in the template without introducing the parent element into the resulting view. This is handy when working with elements that can contain only certain child elements (for example HTML ul and li elements, or select and option elements).

Multiple Root Elements

play_arrow View Demo
<ul> <ng-container *ngIf="isAdmin"> <li>admin-only info 1</li> <li>admin-only info 2</li> </ng-container> <li>public info 1</li> <li>public info 2</li> </ul>
